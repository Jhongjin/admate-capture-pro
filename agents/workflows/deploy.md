# 🚀 자동 배포 워크플로우

코드 수정이 완료되면 아래 단계를 순서대로 실행합니다.

// turbo-all

## 1. 빌드 확인
```
npx next build 2>&1 | Select-Object -Last 20
```
- 빌드 실패 시 에러를 수정한 후 다시 빌드합니다.
- 빌드 성공 시 다음 단계로 진행합니다.

## 2. 변경 파일 스테이징
```
git add -A
```

## 3. 커밋
```
git commit -m "<커밋 메시지>"
```
- 커밋 메시지는 수정 내용을 간결하게 요약합니다.
- 한국어로, `fix:`, `feat:`, `refactor:` 등 prefix를 사용합니다.

## 4. 푸시 (Vercel 자동 배포 트리거)
```
git push origin main
```

## 5. 배포 결과 보고
- 커밋 해시, 변경 파일 수, Vercel 배포 상태를 팀장님께 보고합니다.
